## Q5 契約 v2.3 核心精華（給新 session 背景）

### 命題層

1. **雙軌觸發**：
   - 被動軌（Q4 後用戶主動回訊息 + Q5_intent='continue'）
   - 主動軌（Q4 後 24h 無回應 → cron push）
2. **Q5 訊息不講方案**：不提 4 週/12 週/價格，方案細節留 /apply 頁
3. **/apply 是說服頁**：渴望/可能性/確定感/錨點轉換/報名五章，不是「介紹課程」
4. **真人處理猶豫**：handoff 訊息承諾具體時間 + 不急推課

### 技術層重點

- **pushMessage + Quick Reply** 零擴充（不需擴 pushFlexMessage）
- **Ch.0.8 `performQ5Transition` helper**：atomic UPDATE + race guard + rollback + 48h cron 兜底
- **Ch.0.9 /apply LIFF 驗證**（推方案 A）：擋 URL 分享盜用污染 baseline
- **Ch.5.3 PL/pgSQL RPC**：`submit_application` 含 `IF NOT FOUND RAISE P0002`
- **q5_intent enum**：continue / decline / **ai_failed**（AI 失敗不當 continue）
- **Vercel Pro**：既有 `*/10` cron 確認非 Hobby，加 `maxDuration: 60`
