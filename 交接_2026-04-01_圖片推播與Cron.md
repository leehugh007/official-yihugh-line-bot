## 這次更新了什麼（5 項）

### 1. 推播可以配圖片
推播訊息頂部可以放一張大圖（LINE 的 hero image）。有圖片的推播會自動變成 Flex Message 格式。沒上傳圖片就跟以前一樣。

### 2. 排程文章也可以配圖片
文章排程（Drip）的每一篇都可以加圖片。有圖的文章推送時自動變 Flex Message，沒圖的維持純文字。

### 3. 排程推播更準時
Cron 從每小時改成每 10 分鐘。以前排 20:25 推播最慘要等到 21:00，現在最多延遲 10 分鐘。

### 4. Drip 排程批量發送
排程文章改成 multicast 批量發送（500 人一批），人數增到 1000-2000 也不會 timeout。

### 5. CLAUDE.md 補了完整的資料庫欄位對齊表
所有 Supabase 表的欄位都列在 CLAUDE.md 的「Schema 對齊」區塊。**改程式碼前先對這張表，欄位不在這裡就是不存在。**

## 操作教學

### 推播配圖片

**模板推播：**
1. 後台 → 推播 Tab → 選一個模板 → 點「✏️ 編輯」
2. 往下找到「圖片（選填，顯示在訊息頂部）」
3. 點「點擊上傳圖片」→ 選一張圖
4. 上傳後會看到圖片預覽，LINE 預覽也會更新
5. 如果要移除圖片，點圖片右上角的 ✕
6. 按「儲存」→ 之後每次用這個模板都會帶這張圖
7. 送出時圖片會出現在 LINE 訊息最上方（hero image）

**自訂推播：**
1. 後台 → 推播 Tab → 「+ 建立自訂推播」
2. 寫好訊息 → 往下找到「圖片」→ 上傳
3. 可以同時有圖片 + 按鈕，也可以只有圖片沒按鈕
4. 確認預覽 → 送出

### 排程文章配圖片

1. 後台 → 排程 Tab → 找到要編輯的文章 → 點「✏️ 編輯」
2. 往下找到「圖片（選填）」→ 上傳
3. 按「儲存」
4. 有圖片的文章推送時：圖片在最上面，下面是標題 + 內文 + 按鈕
5. 沒圖片的文章維持原本純文字
6. 編輯列表上會顯示「有圖片」標示

### Supabase 異動

| 異動 | 說明 |

### 圖片上傳流程（完整）

```
後台選檔 → FileReader 轉 Base64
→ POST /api/admin { action: 'upload_image', fileName, fileBase64, contentType }
→ server 端 Buffer.from(base64) → supabase.storage.from('push-images').upload()
→ supabase.storage.getPublicUrl() → 回傳公開 URL
→ 前端把 URL 存進 editData.image_url → 儲存時寫入 DB
→ 推播時 pushFlexMessage({ ..., imageUrl }) → LINE 顯示 hero image
```

### Drip multicast 改動邏輯

**之前**：逐筆 `pushMessage`，每人獨立追蹤連結（帶 userId）。500 人同時到期 = 500 次 API call，會 timeout。

**現在**：按 `step_number` 分群，同一篇文章的用戶一起 `multicastMessage`（500 人一批）。追蹤連結統一用 `drip_{step}` 不帶 userId。

**影響**：Drip 點擊追蹤不再精確到個人（click log 的 line_user_id 可為 null），但 drip_logs 仍記錄「誰收了哪篇」。

### 邊界情況處理

- 有圖片但沒按鈕 → 正常，Flex Message 只有 hero + body，沒 footer
- 沒圖片沒按鈕 → 維持純文字 textMessage
- 有圖片有按鈕 → 完整 Flex：hero + body + footer
- 上傳非圖片格式 → 前端 + 後端雙重擋（只接受 JPG/PNG/WebP）
- 上傳超過 2MB → 前端 + 後端雙重擋
- 排程推播 scheduled push → `handleSendScheduled` 也支援 image_url
