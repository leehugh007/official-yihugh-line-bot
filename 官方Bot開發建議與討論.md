## AI 自動回覆與智慧推播（待討論 💬）

> 2026/4/17 提出

### 背景：目前的限制

目前官方 Bot 收到用戶訊息只能做「關鍵字比對」→ 固定回覆。

| 問題 | 說明 |
|------|------|
| 無法自然對話 | 用戶問的問題不在關鍵字清單內，Bot 就不會回覆或回制式訊息 |
| 不了解用戶 | 用戶聊過什麼、關心什麼、猶豫什麼，系統完全沒有記錄 |
| 無法主動出擊 | 用戶沉默後只能靠排程文章定時推，無法根據個人狀況推送對應內容 |
| 互動率低 | 排程文章是單向推送，缺乏雙向互動機會 |

### 提案：複製 Coach Bot 的 AI 架構

Coach Bot（教練端）已經有一套成熟的 AI 系統，核心架構可以複製到官方 Bot，但內容和邊界需要調整。

**Coach Bot 現有能力：**

| 能力 | 做法 |
|------|------|
| AI 用一休口吻回覆 | Gemini AI + 一休的人設指令（說話風格、核心理念、黃金語錄） |
| 智慧知識庫 | 兩層式：核心理念永遠帶入 + AI 自動判斷話題選對應知識塊 |
| 自動用戶畫像 | 每次對話後自動提取標籤（話題、情緒、核心問題、進度） |
| 對話記錄 | 完整保存每次對話，累積用戶歷程 |

**官方 Bot 要調整的部分：**

| | Coach Bot（教練端） | 官方 Bot（潛在客戶端） |
|------|------|------|
| 對象 | 已報名的學員 | 還沒報名的潛在客戶 |
| 口吻 | 深入指導、解答細節 | 親切引導、激發興趣 |
| 邊界 | 可以教具體方法 | 基本觀念可聊，細節引導報名 |
| 知識庫 | 教練指導主題（21 塊） | 潛在客戶常見問題（約 10-15 塊） |
| 標籤 | 學習進度、情緒、核心問題 | 關注點、猶豫原因、購買意願、痛點 |

### 功能 1：AI 自動回覆（用一休的口吻）

用戶傳任何訊息進來，Bot 都能用一休的風格自然回覆。

**舉例：**
```
用戶：「我試過很多方法都瘦不下來...」

Bot（AI 回覆）：
「我懂那種感覺，其實你不是瘦不下來，
是之前用的方法讓身體一直在對抗你。

很多人跟我說過一樣的話，
後來才發現原來不是意志力的問題，是方法根本就錯了。

你之前都試過什麼方式？我很好奇 😊」
```

**關鍵設計：**
- 基本觀念、心態調整可以自然聊
- 涉及具體方法時，引導到說明會或報名
- 不會一直推銷，而是讓用戶自己產生好奇

### 功能 2：自動用戶畫像

每次對話後，AI 自動分析並記錄用戶狀態。

| 標籤 | 舉例 |
|------|------|
| 關注點 | 體重停滯、產後瘦身、健康檢查紅字（見下方實際數據） |
| 痛點 | 試過節食沒用、運動沒時間、外食族不知道怎麼吃 |
| 猶豫原因 | 價格、怕沒辦法堅持、不確定有沒有效（見下方實際數據） |
| 購買意願 | 高 / 中 / 低（見下方判斷邏輯） |
| 互動頻率 | 活躍 / 偶爾 / 沉默 |

後台用戶 Tab 可直接看到每位用戶的標籤，知道誰對什麼感興趣、誰在猶豫什麼。

**關鍵發現：**
- **問價格 = 最強的成交訊號**（59% 成交率），這些人不是嫌貴，是在做最後確認
- **說「太貴」的人 = 不會報名**（0% 成交率），跟「問價格」完全不同
- **外食族 + 吃過藥的人** 成交率最高，他們試過其他方法、花過錢，最能理解「系統化方法」的價值
- **體脂焦慮 + 復胖** 提及量最大但成交率低，多數只是焦慮但還沒有行動動機
- **提到家人/老公** 的人成交率 50%，代表「幫家人問」或「需要家人同意」的人其實意願很高

**購買意願怎麼判斷？**

不是靠單一條件，而是綜合「行為」+「對話內容」兩個面向來打分：

| 面向 | 高意願訊號 | 中意願訊號 | 低意願訊號 |
|------|-----------|-----------|-----------|
| 行為 | 點擊報名連結、主動問價格/開課時間、多次回覆對話 | 點擊文章連結、偶爾回覆、看過說明會回放 | 只收訊息不點不回、加入後完全沉默 |
| 對話 | 「怎麼報名」「費用多少」「下一期什麼時候」 | 「想了解」「有點興趣」「再考慮看看」 | 「先不用」「只是看看」或完全沒對話 |

**具體做法：AI 每次對話後自動評估，不需要手動判斷**

```
用戶：「費用大概多少？我老公也想一起報名」
                    ↓
AI 分析：提到費用 + 幫家人詢問 = 有具體行動意圖
                    ↓
自動標記：購買意願 → 高
```

```
用戶：「看起來蠻有趣的，但我不確定適不適合我」
                    ↓
AI 分析：表達興趣 + 但有猶豫
                    ↓
自動標記：購買意願 → 中，猶豫原因 → 不確定是否適合
```

意願會隨時間更新——如果一個本來「中」的用戶後來問了價格，就會自動升成「高」。反過來，長期沉默的也會自動降低。

### 功能 3：智慧推播（根據用戶狀態個人化推送）

用戶沉默一段時間後，系統根據標籤自動推送對應內容。

| 用戶狀態 | 推送內容 |
|----------|----------|
| 關注「產後瘦身」+ 2 週沒互動 | 產後學員見證故事 |
| 猶豫「價格」+ 1 週沒互動 | 投資健康 vs 花錢看醫生的觀點文 |
| 購買意願「高」+ 3 天沒互動 | 限時優惠或說明會連結 |
| 點擊過文章但沒回覆 | 根據點擊的文章推送對應的開放式問題（見下方） |

跟排程文章不同，這是**根據每個人的情況客製化推送**。

**「點擊過文章但沒回覆」的具體操作：**

系統知道用戶點了哪篇文章（追蹤連結有記錄），所以可以根據他點的內容，推一則跟那篇文章相關的問題。

| 用戶點擊的文章 | 3 天沒回覆後推送的訊息 |
|--------------|---------------------|
| 瘦瘦針真相 | 「看了瘦瘦針那篇，你之前也有考慮過打針嗎？還是試過其他方法？好奇你的情況 😊」 |
| 溫溫的故事 | 「溫溫的故事有打中你嗎？你現在最想改變的是什麼？」 |
| 代謝重建 vs 節食 | 「看完代謝那篇，有沒有哪個觀念跟你之前想的不一樣？」 |
| 沛蓁的故事 | 「沛蓁的經歷讓你有什麼感覺？你目前的狀況跟她像嗎？」 |
| 體脂計迷思 | 「你平常也會量體脂嗎？很多人看完那篇才發現被數字綁架了」 |

**結合測驗結果更精準：**

系統已經有存每位用戶的代謝類型（`metabolism_type`），5 種類型：

| 代謝類型 | 名稱 | 核心特徵 |
|---------|------|---------|
| highRPM | 高轉速型 | 壓力大、皮質醇高、腹部脂肪 |
| rollerCoaster | 雲霄飛車型 | 血糖不穩、忽高忽低 |
| burnout | 燃燒殆盡型 | 能量透支、節食過度 |
| powerSave | 省電模式型 | 吃很少還瘦不了、代謝降速 |
| steady | 穩定燃燒型 | 基礎不錯但有優化空間 |

所以推送可以做到這種程度：

```
用戶代謝類型：省電模式型
     +
點擊了「代謝重建 vs 節食」但 3 天沒回覆
     ↓
推送：「你是省電模式型，代謝已經降速了，
看完那篇有沒有覺得跟自己很像——吃很少卻瘦不了？

其實第一步不是少吃，而是要敢吃。
聽起來矛盾對吧？想知道怎麼開始的話直接跟我說 😊」
```

```
用戶代謝類型：雲霄飛車型
     +
點擊了「瘦瘦針真相」但 3 天沒回覆
     ↓
推送：「你是雲霄飛車型，血糖不穩才是根本問題。
瘦瘦針那篇看完是不是覺得打針不是長久之計？

你現在一天大概怎麼吃？跟我說一下，我幫你看看 😊」
```

### 功能 3 補充：排程文章 CTA 策略調整

目前排程文章以「給資訊」為主（推文章連結），但多數用戶加入 LINE 前已做過測驗、看過官網，單純給資訊的吸引力有限。

**調整方向：從「給資訊」改成「引發對話」**

不再只是推連結，而是在文章尾端加入開放式問題，引導用戶主動回覆，創造介入的機會。

**6 篇排程文章對應的 CTA：**

| 篇 | 文章主題 | 連結 | CTA（開放式問題） |
|---|---------|------|-----------------|
| 1 | 一休的故事 | /about | 「你目前最困擾的是什麼？體重卡住、復胖、還是不知道怎麼吃？直接跟我說」 |
| 2 | 瘦瘦針真相 | /articles/ozempic-cost-and-regain | 「你之前試過什麼方法？歡迎跟我分享，我幫你看看問題在哪」 |
| 3 | 溫溫的故事 | /articles/wenwen-mom-story | 「你也有類似的經歷嗎？回覆告訴我，我來幫你看看」 |
| 4 | 代謝重建 vs 節食 | /articles/metabolism-reset-vs-dieting | 「你現在一天大概怎麼吃？簡單說就好，我幫你看看有沒有可以調整的地方」 |
| 5 | 沛蓁的故事 | /articles/chicken-soup-love-story | 「如果你也想開始改變，最想先改善的是什麼？」 |
| 6 | 體脂計迷思 | /articles/body-fat-scale-liar | 「最近有一場免費線上說明會，一休會分享怎麼開始。想了解的話回覆『說明會』」 |

前 5 篇用**開放式問題**引發對話，用戶回的內容不觸發關鍵字，一休可以手動接話做銷售。第 6 篇導向說明會做轉換。

**為什麼這樣做：**
- 用戶回覆後，AI 自動回覆可以接住對話（搭配功能 1）
- 對話過程中自動提取用戶畫像（搭配功能 2）
- 沉默用戶的再推播也能更精準（搭配功能 3）
- 一休也可以看到對話紀錄，需要時手動跟進高意願用戶

### 跟現有排程文章的關係

三者互補，不衝突：

```
排程文章（現有）     →  定時推送，建立品牌認知，人人一樣
AI 自動回覆（新增）  →  即時互動，解答疑問，個人化
智慧推播（新增）     →  主動出擊，根據個人狀態推送，精準觸達
```

排程文章負責「廣撒」，AI 回覆負責「接住」，智慧推播負責「追回」。

### 建議開發順序

| 階段 | 內容 | 可獨立上線 |
|------|------|-----------|
| Phase 1 | AI 自動回覆：關鍵字沒命中 → 轉 AI 回覆，建對話紀錄表 | ✅ |
| Phase 2 | 用戶畫像：每次對話自動提取標籤，後台顯示標籤 | ✅ |
| Phase 3 | 智慧推播：cron 掃描沉默用戶，根據標籤選推播內容 | ✅ |

### 技術面與成本

| 項目 | 說明 |
|------|------|
| AI 模型 | Gemini Flash Lite（跟 Coach Bot 相同） |
