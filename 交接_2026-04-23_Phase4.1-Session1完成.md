## Session 1 實戰學到

### 技術決策

- **HMAC 取代 LIFF**：之前 5 輪 yi-challenge 定的 LIFF 方案 A，PR #26/#30 實證做不到。修法是 Session 1 的核心價值（契約 v2.3 → v2.4）。
- **Secret 走 env + key version rotate**（方案 C）：不走 DB，避免每次 verify 50-200ms 延遲；支援 rotate 雙驗期 ≥25h。
- **Max_age 走 env 不走 settings**：Feasibility M1 找出的熱路徑問題，改 env 避免 verify 打 DB。
- **Shape whitelist 是命脈**：Adversarial C1/C2 找出 payload injection 攻擊路徑（`trigger=passive&userid=U_victim`），verify 入口必須白名單擋下。
- **POST body not query**：擋 array injection + parser 行為一致性。

### 流程決策

- **yi-challenge 雙 agent 平行跑**：adversarial（opus）+ feasibility（opus）一起跑效率高，adversarial 找安全漏洞、feasibility 找實作可行性。
- **一個 commit 一個概念**：5 個 commit 分（db / settings / helper / routes / docs），review 容易。
- **MCP apply_migration + repo .sql 檔雙寫**：DB 實時更新 + repo 留部署紀錄，順序「先寫檔 → git commit → MCP 執行」。
