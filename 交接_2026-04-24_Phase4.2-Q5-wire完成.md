## 技術架構（Phase 4.2 後）

### 三條路都通 Q5 軟邀請（Ch.5.1a 被動軌 + Ch.5.1b 主動軌）

```
stage=4
   ├─ 用戶自由文字 ──→ classifyQ5Intent AI
   │                    ├─ continue → performQ5Transition(passive) + pushQ5SoftInvite
   │                    ├─ decline  → updateQ5Intent 靜默
   │                    └─ ai_failed→ updateQ5Intent 保持 stage=4（cron 也排除）
   │
   ├─ 用戶按 Q4 Quick Reply「想聽聽」(postback q4_continue)
   │     → performQ5Transition(passive) + pushQ5SoftInvite（跟上面 continue 等價）
   │
   └─ Q4 後 24h 無動作 ──→ cron q5-maintenance 主動軌
                            → performQ5Transition(active) + pushQ5SoftInvite(active)
```

### stage=6 → 7（用戶點 /apply）
```
Quick Reply「看看做法」→ /apply?userid=X&sig=...
/apply LIFF init → POST /api/apply/visit → verifyQ5ApplySig → stage 6→7 + q5_click_count++
```

### stage=7 → 8（用戶填表）
```
/apply 表單 → POST /api/apply/submit → verifyQ5ApplySig + shape check
→ submit_application RPC → INSERT applications + UPDATE stage=8
→ cron q5-maintenance 段 4 每小時掃 pending 通知一休+婉馨
```

### stage=6 無反應 → cron reset 48h
```
stage=6 + q5_sent_at < now()-48h → atomic UPDATE stage=6→4 + 清 q5_sent_at
→ 下輪主動軌 cron 再試一次（若仍符合條件）
```

### stage=7 點了沒填表 → cron visit-followup 24h
```
stage=7 + q5_clicked_at < now()-24h + q5_visit_followup_sent_at IS NULL
→ pushMessage(q5_visit_followup_text)（1 次就好，不重推）
```

## 關鍵設計決策

1. **callGemini 共用 vs 複製** → 選共用（export from ai-classifier.js）。`__test` 早已暴露，改成正式 export 是 minimal change。避免複製 fetch/abort 邏輯。

2. **source='passive' for q4_continue button** → 用戶主動觸發 = 被動軌定義（跟 stage=4 自由文字 continue 等價）。active 專指 cron 24h 無回應場景。

3. **失敗路徑靜默（不 fallback 訊息）** → race_lost / push_failed_rollback 都不回 fallback，cron 會接手。避免雙重訊息污染 yi-voice。

4. **5 error path 全 fallback ai_failed** → 即使 gemini_no_key（環境問題）也 return ai_failed + console.error CRITICAL（TODO Phase 4.3 接 telegram）。

5. **cron 合併一路不拆** → drip 132 人歷史 <45s，合併 4 段目前估計 <45s 安全。若未來量變大 >45s 再拆（契約 Ch.4.3 backup plan）。

6. **q5_test_mode_cron=true 排除 TEST_ALLOWLIST** → 主動軌 + visit-followup 跳過一休+婉馨。保護 dev 帳號不被 cron 自動推。

7. **concurrency 10（比 drip 20 保守）** → AI cost 每個 classifier 打一次 Gemini，且 LINE rate limit 顧慮。

8. **getUserPathState SELECT 擴充順便修幽靈 guard** → L220 讀 `state?.q5_sent_at` 過去永遠 undefined，非文字 pre-check 沒生效。Phase 4.2 順手修。
